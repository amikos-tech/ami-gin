<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Merge callback -> recover helper | Internal merge code may panic after validator approval; the helper converts that internal invariant failure into a terminal builder error. |
| Recover helper -> logger seam | Panic metadata crosses into caller-configured logging; attrs must not contain raw document content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T16-01 | Tampering | document bookkeeping after recovered merge panic | mitigate | Task 2 sets `b.tragicErr` and returns before doc bookkeeping when `runMergeWithRecover` returns an error. |
| T16-02 | Denial of Service | panic during `mergeStagedPaths` | mitigate | Task 2 wraps only `mergeStagedPaths` in `runMergeWithRecover`, converts panic to error, and tests direct and builder-level refusal paths. |
| T16-03 | Information Disclosure | recovered panic logging | mitigate | Task 2 logs only `error.type` and `panic_type`; tests assert no `panic_value` key and no raw secret string in attrs. |
| T16-04 | Tampering | marker/signature invariant | transfer | Plan 16-04 wires the static marker check into local lint and CI. |
</threat_model>
