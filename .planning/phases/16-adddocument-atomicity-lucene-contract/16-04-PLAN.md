    Update `Makefile`:

    - add `.PHONY: check-validator-markers`
    - add a `check-validator-markers` target that scans root `*.go` files with POSIX-compatible `awk`
    - Addresses review concern: the check must positively enforce marker presence, not only reject marked functions that return `error`.
    - the check must:
      - detect marker lines with whitespace tolerance using the equivalent of `^[[:space:]]*//[[:space:]]*MUST_BE_CHECKED_BY_VALIDATOR[[:space:]]*$`
      - require the marker line to directly precede a `func ` declaration; if the next line is blank, another comment, or any non-function line, fail non-zero and print `MUST_BE_CHECKED_BY_VALIDATOR marker must directly precede function declaration:`
      - inspect the directly following function signature
      - fail non-zero and print `MUST_BE_CHECKED_BY_VALIDATOR function returns error:` when the marked signature contains `error` in its return type
      - collect marked function names and fail if any marked function is not one of `mergeStagedPaths`, `mergeNumericObservation`, or `promoteNumericPathToFloat`
      - fail if any of the three expected functions is absent from the marked-function set, printing `missing MUST_BE_CHECKED_BY_VALIDATOR marker for mergeStagedPaths`, `missing MUST_BE_CHECKED_BY_VALIDATOR marker for mergeNumericObservation`, or `missing MUST_BE_CHECKED_BY_VALIDATOR marker for promoteNumericPathToFloat`
      - fail if the final marked-function count is not exactly 3
      - clear the marker after inspecting the function signature
    - wire `lint` to run this check by changing the target to `lint: check-validator-markers` before `golangci-lint run`
    - update the `help` text for `lint` to mention marker checks and golangci-lint

    Use POSIX-compatible `awk`; do not add a custom golangci-lint plugin or Go analyzer.
