Introduce the observability foundation for Phase 14 without yet changing the public query/build/serialize boundaries. This plan creates the repo-owned logging and telemetry seams, wires them into `GINConfig`, installs silent defaults, and ships the `slog` / stdlib adapters as opt-in subpackages.

Purpose: establish the single logging convention, the local `Signals` container, and the frozen vocabulary source before any boundary migration work begins. The rest of the phase depends on this surface being stable.

Output: new `logging/` and `telemetry/` packages, additive `GINConfig` fields and options, root-module-safe OTel API/noop dependencies only, and tests that lock silent defaults and adapter behavior.
</objective>

    Create a new top-level `logging` package with four files:

    1. `logging/logger.go`
       - define `type Level uint8` with `LevelDebug`, `LevelInfo`, `LevelWarn`, `LevelError`
       - define a repo-owned `Attr` struct and helper constructors for the frozen vocabulary keys:
         `AttrOperation`, `AttrPredicateOp`, `AttrPathMode`, `AttrStatus`, `AttrErrorType`
       - define `type Logger interface { Enabled(Level) bool; Log(Level, string, ...Attr) }`
       - add small helpers `Debug`, `Info`, `Warn`, `Error` that normalize nil loggers through `Default(...)`
       - `Debug` MUST guard on `Enabled(LevelDebug)` before calling `Log`; `Info`/`Warn`/`Error` may delegate directly

    2. `logging/attrs.go`
       - freeze the canonical keys in a single source file:
         `operation`, `predicate_op`, `path_mode`, `status`, `error.type`
       - define the only allowed `path_mode` string labels in the same source of truth: `exact`, `bloom-only`, `adaptive-hybrid`
       - provide a normalization helper for `error.type` that collapses unknown values to `other`
       - do not define constructors for path names, predicate values, doc IDs, row-group IDs, or term IDs

    3. `logging/noop.go`
       - provide a shared noop logger plus `NewNoop()` and `Default(logger Logger) Logger`
       - noop logger must report `Enabled(...) == false`

    4. `logging/doc.go`
       - document the context-free contract, the adapter split, and the safe-metadata restrictions
       - explicitly state that INFO-level attrs are limited to the frozen allowlist and that trace IDs are not injected into logs

    Keep the payload contract typed (`...Attr`), not variadic `...any`.
  </action>

    - `logging/attrs.go` defines `path_mode`'s bounded emitted values as `exact`, `bloom-only`, and `adaptive-hybrid`.
    - `logging/noop.go` contains `func NewNoop() Logger` and `func Default(logger Logger) Logger`.
