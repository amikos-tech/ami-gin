<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSONL input -> CLI report | Untrusted line values may be copied verbatim into text/JSON samples. |
| CLI failure accounting -> row-group positions | Failed lines must not consume accepted-document row-group positions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T18-08 | Information Disclosure | CLI failure samples | accept | D-08 requires verbatim values. The CLI does not redact or truncate; report growth is bounded by 3 samples per layer and docs warn callers about disclosure. |
| T18-09 | Tampering | accepted document positions | mitigate | Tests assert row groups derive from accepted documents only; failed lines use separate `input_index`. |
| T18-10 | Denial of Service | large failure streams | mitigate | Group sample count cap prevents unbounded sample accumulation while counts still accumulate. Large individual values remain accepted per D-08. |
| T18-11 | Repudiation | JSON/text ordering | mitigate | Deterministic group ordering is specified and tested to make diagnostics stable. |
</threat_model>
