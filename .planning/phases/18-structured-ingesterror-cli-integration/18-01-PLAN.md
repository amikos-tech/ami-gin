<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller document bytes -> public error API | Untrusted document content may be copied verbatim into `IngestError.Value`. |
| Parser/sink callback -> builder classification | Stage callback errors must keep their original numeric/transformer/schema layer instead of becoming parser failures. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T18-01 | Information Disclosure | `IngestError.Value` | accept | D-08 requires verbatim value capture and caller-owned redaction. No truncation is applied in this phase; docs and changelog must state this clearly. |
| T18-02 | Tampering / Repudiation | ingest failure classification | mitigate | Tasks 2 and 3 add per-layer `errors.As` tests and preserve stage callback provenance before parser wrapping. |
| T18-03 | Denial of Service | large error values | accept | No byte cap is applied to preserve D-08 verbatim semantics. CLI output in Plan 18-03 is bounded by sample count; callers logging library errors own output policy. |
| T18-04 | Elevation of Privilege | parser contract errors | mitigate | Parser implementation contract failures remain non-`IngestError` and hard regardless of parser failure mode. |
</threat_model>
