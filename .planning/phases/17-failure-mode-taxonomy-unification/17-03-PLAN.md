<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller JSON bytes -> parser/staging | Untrusted document content can fail parsing, transformer conversion, or numeric classification. |
| Staged document state -> durable builder indexes | Soft skips must return before staged observations mutate shared path/index/bookkeeping state. |
| Internal merge invariant -> tragic builder state | Validator-missed merge panics must close the builder rather than being treated as caller-configured soft skips. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering / Repudiation | parser/numeric/transformer defaults | transfer | Plan 17-01 pins hard defaults and invalid-value validation; this plan consumes normalized modes only. |
| T-17-02 | Tampering | `AddDocument`, staging methods, `commitStagedPaths` | mitigate | Tasks 1-2 route soft failures to a nil return before durable merge/bookkeeping and add state, query, and encoded-byte oracle tests. |
| T-17-03 | Tampering / Denial of Service | `runMergeWithRecover` | mitigate | Task 1 explicitly does not soften recovered merge panics and adds `TestNumericFailureModeSoftKeepsMergeRecoveryTragic`. |
| T-17-04 | Tampering | serialized transformer metadata | transfer | Plan 17-02 owns v9 wire compatibility; this plan only changes ingest-time routing. |
| T-17-05 | Information Disclosure | soft failure handling | mitigate | Tasks add no logging or telemetry for soft skips and keep fixed test payloads out of runtime logs. |
