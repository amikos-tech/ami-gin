<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller config -> builder ingest routing | Untrusted or mistaken caller-supplied mode strings influence whether document failures are rejected or skipped. |
| Public API -> serialized transformer metadata | Public mode values differ from legacy transformer wire tokens; normalization must not change finalized index format. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering / Repudiation | `GINConfig`, `DefaultConfig`, `WithParserFailureMode`, `WithNumericFailureMode` | mitigate | Tasks 1-2 default every layer to `IngestFailureHard`, normalize empty modes to hard, and reject invalid values at option/config validation time. |
| T-17-02 | Tampering | `builder.go` soft-skip routing | transfer | Plan 17-03 implements whole-document soft skips before durable mutation and verifies no partial indexing. |
| T-17-03 | Tampering / Denial of Service | `runMergeWithRecover` | transfer | Plan 17-03 explicitly keeps recovered merge panics outside soft-mode handling. This plan does not modify recovery routing. |
| T-17-04 | Tampering | `TransformerSpec.FailureMode`, serialized config | transfer | Plan 17-02 preserves v9 transformer wire tokens and legacy decode compatibility. This plan creates the normalized API type consumed there. |
| T-17-05 | Information Disclosure | failure-mode handling paths | mitigate | This plan adds no logging, telemetry, or raw input formatting for soft failures. |
</threat_model>
