  <files>cmd/gin-index/experiment.go, cmd/gin-index/experiment_output.go</files>

  </read_first>
  <action>
    Extend the experiment ingest loop with two final flags:

    - `--sample N`
    - `--on-error continue|abort`

    Implement them with these exact rules:

    - reject `--sample < 0`
    - default `--on-error` to `abort`
    - reject any `--on-error` value other than `continue` or `abort`
    - when `--sample N` is greater than `0`:
      - skip the Plan 15-01 pass-1 counting/spooling path for both file and stdin inputs
      - initialize the builder with `max(1, ceil(N/rgSize))`
      - stream directly from the original source
      - stop after `N` successful ingests or EOF
    - when `--sample` is `0`, preserve the Plan 15-01 two-pass sizing path
    - maintain counters:
      - `processed_lines`
      - `documents` (successfully ingested documents)
      - `skipped_lines`
      - `error_count`
    - sample limit applies to `documents`, not `processed_lines`
    - blank lines follow the Plan 15-01 invalid-input rule and participate in abort/continue behavior like any other malformed record
    - when `--on-error continue`:
      - print `line N: <error>` to `stderr`
      - increment `skipped_lines` and `error_count`
      - continue to the next line
    - when `--on-error abort`:
      - print `line N: <error>` to `stderr`
      - return a non-zero exit code immediately

    Surface the counters in both text mode and JSON mode summary output.
    Compute summary and predicate row-group totals from `usedRowGroups`, not `idx.Header.NumRowGroups`, so skipped or unsampled capacity never inflates pruning ratios.
    If zero documents ingest, emit a coherent zero-document summary in both text and JSON mode rather than panicking.
  </action>

    - summary and predicate row-group math use used row groups rather than allocated capacity.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller-provided JSONL → line error handling | Malformed input must either fail fast or continue deterministically, with line numbers preserved. |
| Summary counters → user/CI interpretation | Text and JSON summaries become operator evidence; counter drift is a correctness bug. |
| Repo code / go.mod → CLI charter | Dependency or UI-surface drift can silently change the tool's operational shape without failing build tests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-07 | Denial of Service | Continue-mode error tracking | mitigate | Stream diagnostics directly to `stderr` and retain only counters, never an unbounded in-memory error list. |
| T-15-08 | Repudiation | Malformed-line reporting | mitigate | Include exact `line N:` diagnostics in both abort and continue paths; lock them with end-to-end tests. |
| T-15-09 | Tampering | CLI charter drift | mitigate | Add `experiment_policy_test.go` guards for forbidden dependencies, imports, ANSI/TTY logic, and parser-flag exposure. |

ASVS posture: V5 (Input Validation) and V14 (Configuration) apply here through explicit flag validation, line-numbered failure reporting, and executable policy guards against unreviewed CLI surface growth.
</threat_model>
