  <action>
    Implement the experiment foundation in `cmd/gin-index/experiment.go` with these exact rules:

    - reject `--rg-size <= 0` before ingest begins
    - reject an input path that resolves to a directory; `experiment` accepts exactly one JSONL file or `-`
    - keep counting-pass and ingest-pass line semantics identical:
      - wrap readers in `bufio.NewReader(...)`
      - read one line at a time with `ReadBytes('\n')`
      - accept a final JSON document without a trailing newline
      - do not synthesize an extra record for a terminal newline
      - treat blank JSONL lines as invalid input rather than silently skipping them
    - implement a two-pass ingest contract because `GINBuilder` needs `numRGs` up front:
      - local-file input: first pass counts candidate JSONL records, second pass performs the real build
      - stdin input: first pass spools stdin into a temp file while counting candidate JSONL records, closes the temp file, reopens it for the real build, and removes it with `defer os.Remove(...)` after the build pass
    - in the real ingest pass, trim only trailing `\n` and `\r\n`

    Build the index with `gin.NewBuilder(gin.DefaultConfig(), estimatedRGs)` where:
    - `estimatedRGs` is `max(1, ceil(candidateRecordCount/rgSize))`
    - synthetic row-group id is `ingestedDocs / rgSize`
    - `DocID` passed to `AddDocument(...)` is `gin.DocID(rgID)`
    - track `usedRowGroups` separately from builder capacity because `idx.Header.NumRowGroups` reflects allocated RG slots:
      - text summary's `Row Groups` field must print `usedRowGroups = ceil(ingestedDocs/rgSize)` or `0` when no documents ingest
      - do not report `estimatedRGs` to users

    Text output must be:

    1. `Experiment Summary:` block containing at minimum:
       - `Input`
       - `Documents`
       - `Row Groups`
       - `RG Size`
    2. the existing per-path table via `writeIndexInfo(stdout, idx)`

    Do not implement `--json`, `--test`, `-o`, `--sample`, or `--on-error` behavior in this plan; this plan only establishes the foundation text path.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI input path/stdin → JSONL ingest | The command reads caller-controlled bytes from either a local file or stdin and feeds them into the real builder path. |
| JSONL line buffer → builder | Each line becomes one attempted `AddDocument(...)` call; malformed lines must not cause silent truncation or a whole-file memory spike. |
| Summary renderer → stdout/stderr | User-facing results go to `stdout`; diagnostics and parse/ingest failures go to `stderr`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Denial of Service | Oversized JSONL lines | mitigate | Use `bufio.Reader.ReadBytes('\n')` rather than default `bufio.Scanner`; add `TestRunExperimentLargeLineNoTruncation` with a >64 KiB line. |
| T-15-02 | Tampering | Invalid row-group size | mitigate | Reject `--rg-size <= 0` before reading input; add `TestRunExperimentRejectsInvalidRGSize`. |
| T-15-04a | Denial of Service | stdin cannot satisfy builder RG sizing in one pass | mitigate | Explicit temp-file spooling for stdin in the counting pass; no full-file in-memory buffering. |
| T-15-04b | Denial of Service | leaked stdin spool files | mitigate | Close and remove the temp spool file after the build pass with `defer os.Remove(...)`; keep stdin support covered by direct tests. |
| T-15-03 | Information Disclosure | Mixed result/diagnostic streams | mitigate | Keep the summary and path table on `stdout` only; reserve `stderr` for parse and ingest failures. Tests assert text output shape separately from failure cases. |

ASVS posture: V5 (Input Validation) applies at the CLI boundary through flag validation and explicit large-line handling. V10 (Malicious Input Handling) is partially addressed here by bounded-memory ingest and failure separation.
</threat_model>
