    Extend `runExperiment(...)` with four new flags:

    - `--json`
    - `--test`
    - `-o`
    - `--log-level off|info|debug`

    Implement them with these exact rules:

    - `--json`
      - render via `writeExperimentJSON(...)`
    - `--test`
      - parse with the existing `parsePredicate(...)`
      - evaluate with `idx.EvaluateContext(context.Background(), []gin.Predicate{pred})`
      - compute totals against the report summary's used row-group count, not `idx.Header.NumRowGroups`
      - compute:
        - `matched = len(result.ToSlice())`
        - `pruned = summary.row_groups - matched`
        - `pruning_ratio = float64(pruned) / float64(summary.row_groups)` when used row groups > 0, else `0`
      - never print RG ID lists
    - `-o`
      - reject any output path that does not end with `.gin`
      - encode with `gin.Encode(idx)`
      - write via `writeLocalIndexFile(...)`
      - preserve source file permissions for local-file inputs via `localOutputMode(inputPath)`
      - use `defaultLocalArtifactMode` when the input source is stdin
    - `--log-level`
      - `off`: leave the default silent config
      - `info` / `debug`: build a repo-owned logger with `logging/slogadapter`
      - route logging output to `stderr`
      - do not add colour or TTY detection

    The command must still keep user-facing report output on `stdout`.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Index/report model → JSON output | The command turns index metadata into a machine-consumed schema; drift here is an integration break even if text mode still looks fine. |
| Predicate string → query evaluation | Caller-controlled predicate text is parsed and evaluated against the built index. |
| Index bytes → local output file | The command writes encoded index bytes to a caller-selected local path. |
| CLI log-level → stderr diagnostics | Log-level wiring must not mix library diagnostics into stdout report output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Information Disclosure | CLI logging surface | mitigate | Use the repo-owned logging seam and the existing frozen attr policy; do not print raw predicate values or RG IDs in log output. |
| T-15-05 | Tampering | JSON schema drift | mitigate | Add `TestRunExperimentJSONGolden` and explicit struct-based marshaling; do not use unordered maps. |
| T-15-06 | Tampering | Sidecar output unreadable or permission-incorrect | mitigate | Require `-o` to end with `.gin`, reuse `writeLocalIndexFile(...)`, derive output mode from the input source, and lock roundtrip behavior in `TestRunExperimentWritesSidecarRoundTrip`. |

ASVS posture: V7 (Error Handling) applies to parse and output failures; V9 (Data Protection) applies to keeping observability output and predicate reporting count-focused rather than value-focused.
</threat_model>
