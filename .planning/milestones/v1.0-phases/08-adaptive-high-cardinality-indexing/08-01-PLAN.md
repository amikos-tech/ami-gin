  truths:
    - High-cardinality string paths can end Phase 08 in three explicit modes: full exact, bloom-only, or adaptive-hybrid.
    - Adaptive-hybrid paths promote only bounded hot terms to exact row-group bitmaps using row-group coverage, not raw document counts.
    - Non-promoted terms on adaptive paths never fall back to `AllRGs()` after a positive bloom hit; they use deterministic bucket bitmaps and remain false-negative-free.
    - `NE` and `NIN` do not invert lossy bucket results on adaptive paths; they stay conservative unless the queried term is exact-promoted.

  key_links:
    - `pathBuildData.stringTerms map[string]*RGSet` remains the source of truth for promotion and bucket construction.
    - Adaptive bucket lookup is deterministic and driven by the same hash path in build and query code.
    - String-length stats remain a secondary filter only and do not replace bucket fallback.

<objective>
Introduce the core adaptive-hybrid indexing behavior for high-cardinality string paths.

Purpose: replace the current exact-or-bloom-only cliff with a bounded hybrid layout that keeps exact pruning for hot values and conservative compact fallback for the tail.
Output: adaptive path mode/types in memory, finalize-time promotion and bucket assembly, query semantics for positive and negative membership operators, and regression/property coverage for no-false-negative behavior.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| per-term exact coverage -> promotion selection | Promotion mistakes can waste size or degrade pruning quality |
| term hash -> bucket bitmap | Incorrect or unstable hashing can drop matching row groups |
| adaptive query path -> negative predicate logic | Inverting a lossy bucket result can create false negatives |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-08-02 | D | promoted exact term set | mitigate | Enforce row-group threshold, coverage ceiling, and promoted-term cap. |
| T-08-03 | T/I | adaptive negative predicates | mitigate | Keep `NE` / `NIN` exact-only for promoted terms and conservative otherwise. |
