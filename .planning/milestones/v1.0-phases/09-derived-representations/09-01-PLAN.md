<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source path -> companion registrations | overwrite or duplicate aliases can silently drop intended companions |
| raw prepared source value -> sibling companions | accidental chaining can change semantics based on registration order |
| companion failure -> transactional builder merge | silent fallback or partial merge would corrupt the feature contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | T/I | config registration | mitigate | Reject duplicate aliases per canonical source path and require explicit alias strings. |
| T-09-02 | T/I | builder fan-out | mitigate | Apply every companion to the same raw prepared source value; no transform chaining. |
| T-09-03 | T/R | strict failure semantics | mitigate | Convert companion derivation failure into explicit `AddDocument()` errors before merge. |
</threat_model>
