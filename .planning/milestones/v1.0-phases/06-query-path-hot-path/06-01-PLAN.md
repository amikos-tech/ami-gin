<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public query/config paths -> canonicalization helper | Untrusted caller-supplied JSONPath strings must be validated before they influence lookup or config binding |
| builder traversal paths -> persisted `PathDirectory` | Internal traversal strings become durable path identity used by all future queries |
| decoded `PathDirectory` bytes -> derived lookup state | Deserialized path names must not silently collide after canonicalization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | T | query/config path canonicalization | mitigate | Use `canonicalizeSupportedPath(path string) (string, error)` for public query/config inputs and keep unsupported forms rejected at the validation boundary. |
| T-06-02 | T/R | duplicate canonical names during rebuild | mitigate | `rebuildPathLookup()` returns `ErrInvalidFormat` on canonical collisions and `Decode()` propagates that error instead of overwriting `pathLookup`. |
| T-06-03 | D | `findPath()` hot path | mitigate | Replace linear scans with `pathLookup` and keep missing/unsupported paths on the existing `AllRGs()` fallback path. |
| T-06-04 | E | mutable derived lookup state | mitigate | Document `GINIndex` immutability and rebuild `pathLookup` only inside `Finalize()` and `Decode()`, never lazily during query execution. |
</threat_model>
