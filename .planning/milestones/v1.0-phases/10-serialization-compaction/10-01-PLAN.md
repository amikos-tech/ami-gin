<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ordered string payload -> decoded path/term order | any reorder corrupts bitmap-to-term binding or `PathID`-to-metadata binding |
| mode marker -> decoder branch | unknown or ambiguous markers can misparse the wire payload |
| v8 payload -> v9 decoder | silent compatibility would undermine the explicit rebuild-only contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | T/I | ordered-string decoding | mitigate | Preserve exact caller order through explicit order-preserving helpers and validate decoded counts. |
| T-10-02 | T/I | section-mode dispatch | mitigate | Use explicit `compactStringModeRaw` / `compactStringModeFrontCoded` markers and reject unknown values. |
| T-10-03 | R | wire compatibility | mitigate | Bump to v9 explicitly and keep strict version mismatch rejection. |
</threat_model>
