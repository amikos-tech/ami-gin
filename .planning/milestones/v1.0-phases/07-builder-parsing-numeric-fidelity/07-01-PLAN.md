<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw JSON bytes -> explicit parser | Untrusted document bytes must not silently coerce unsupported numerics into rounded values |
| per-document staging -> global builder state | A document that fails validation must not partially mutate global indexes or counters |
| transformed subtree -> numeric classifier | Transformer outputs must go through the same numeric safety checks as raw JSON values |
| encoded numeric metadata -> decoded query behavior | Encode/decode must preserve int-only vs float-or-mixed semantics exactly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | T | raw numeric input | mitigate | Parse numeric literals explicitly from source text and reject unsupported/lossy forms with path-aware errors. |
| T-07-02 | T/I | mixed-path promotion | mitigate | Promote from int-only to float-or-mixed only when every integer is within the exact float64 range `[-(1<<53), +(1<<53)]`; otherwise fail the document. |
| T-07-03 | T/R | builder mutation ordering | mitigate | Stage document-local changes and merge only after parse/validation success so failures cannot partially index a document. |
| T-07-04 | R | encode/decode numeric mode | mitigate | Persist exact-int metadata and add decode-parity tests for int-only paths and transformer-driven numeric paths. |
| T-07-08 | D | wide-document staging | mitigate | Keep `documentBuildState` path-sparse, materialize only transformer-targeted subtrees, and verify wide-document cost separately in Plan `07-02` instead of retaining a full-document mirror. |
</threat_model>
