**Archived:** 2026-04-27
**Original defined:** 2026-04-23
**Status:** complete

## Requirements

### Atomicity (Lucene contract)

- [x] **ATOMIC-01**: `AddDocument` returning a non-tragic error leaves the builder in a state indistinguishable from never having received the failed call. Completed by 16-03.
- [x] **ATOMIC-02**: `mergeStagedPaths` and `mergeNumericObservation` become infallible; `validateStagedPaths` fully simulates merge failure modes against real `pathData`. Completed by 16-01 and 16-04.
- [x] **ATOMIC-03**: `tragicErr` is reserved for internal-invariant violations only; user-input failures do not reach it. Completed by 16-02 and 16-03.

### Failure-Mode Taxonomy

- [x] **FAIL-01**: Unified `IngestFailureMode` replaces the transformer-only failure-mode constants and extends to parser and numeric-promotion layers. Completed by Phase 17.
- [x] **FAIL-02**: `WithParserFailureMode` and `WithNumericFailureMode` were added; hard remains the default and soft skips failed documents. Completed by Phase 17.

### Structured IngestError

- [x] **IERR-01**: Exported `IngestError` carries `Path`, `Layer`, `Cause`, and verbatim `Value`, with `errors.As` extraction coverage. Completed by Phase 18 Plan 01.
- [x] **IERR-02**: Ingest-error sites wrap underlying hard document failures in `IngestError`, with behavior and AST guard coverage. Completed by Phase 18 Plans 01-02.
- [x] **IERR-03**: `gin-index experiment --on-error continue` reports per-layer grouped failures with structured samples in text and JSON modes. Completed by Phase 18 Plan 03.
