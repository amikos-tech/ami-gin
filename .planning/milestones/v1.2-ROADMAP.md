# Milestone v1.2: Ingest Correctness & Per-Document Isolation

**Status:** SHIPPED 2026-04-27
**Phases:** 16-18
**Total Plans:** 12

## Overview

v1.2 brought `AddDocument` in line with a per-document ingest contract: ordinary bad input leaves the builder consistent and reusable, while only internal invariant failures enter a terminal tragic state. The milestone also unified ingest failure configuration across parser, transformer, and numeric layers, then exposed structured per-document failures through `IngestError` and the experiment CLI.

## Phases

### Phase 16: AddDocument Atomicity (Lucene contract)

**Goal:** `AddDocument` returning a non-tragic error leaves the builder in a state indistinguishable from never having received the failed call.
**Depends on:** Nothing
**Requirements:** ATOMIC-01, ATOMIC-02, ATOMIC-03
**Plans:** 4 plans

Plans:

- [x] 16-01: Validator-complete numeric promotion hoisting and no-error merge signatures
- [x] 16-02: Tragic terminal state rename, merge recovery, and safe recovery logging
- [x] 16-03: Public failure catalog and AddDocument atomicity property tests
- [x] 16-04: Local lint and CI marker/signature enforcement

Outcome:

- `mergeStagedPaths`, `mergeNumericObservation`, and `promoteNumericPathToFloat` are validator-backed and infallible by signature.
- `poisonErr` was renamed to `tragicErr` and narrowed to internal-invariant / recovered-panic paths.
- A 1000-document property test proves full attempted corpora and clean-only corpora encode identically when ordinary document failures are interleaved.
- Local and CI marker checks enforce the validator-before-merge contract.

### Phase 17: Failure-Mode Taxonomy Unification

**Goal:** Provide one mental model for document ingest failures across parser, transformer, and numeric-promotion layers.
**Depends on:** Phase 16
**Requirements:** FAIL-01, FAIL-02
**Plans:** 4 plans

Plans:

- [x] 17-01: Unified public failure-mode API and parser/numeric config defaults
- [x] 17-02: v9 transformer failure-mode serialization compatibility
- [x] 17-03: Parser, transformer, and numeric soft-skip routing with atomicity tests
- [x] 17-04: Breaking-change changelog note and hard-vs-soft example

Outcome:

- Public callers now use `IngestFailureMode` with `IngestFailureHard` and `IngestFailureSoft`.
- `WithParserFailureMode` and `WithNumericFailureMode` were added; parser and numeric defaults remain hard.
- Transformer v9 metadata compatibility was preserved through private `strict` / `soft_fail` wire-token mapping.
- Soft mode skips whole failed documents without durable partial state or row-group holes.

### Phase 18: Structured `IngestError` + CLI integration

**Goal:** Make per-document failures actionable by callers through structured errors and CLI reporting.
**Depends on:** Phase 16, Phase 17
**Requirements:** IERR-01, IERR-02, IERR-03
**Plans:** 4 plans

Plans:

- [x] 18-01: Public `IngestError` API, helper formatting, and builder hard-failure wrapping
- [x] 18-02: Behavior matrix and focused AST enforcement for hard ingest sites
- [x] 18-03: Experiment CLI grouped failure aggregation, deterministic output, and 100-line fixture
- [x] 18-04: Public docs, changelog note, and final verification

Outcome:

- Exported `IngestError` carries `Path`, `Layer`, `Err`, and verbatim `Value`.
- Hard parser, transformer, numeric, schema, and validator-replayed failures are `errors.As` friendly.
- Parser contract errors, tragic/internal errors, and soft skips remain outside `IngestError`.
- `gin-index experiment --on-error continue` reports grouped failures in text and JSON, with deterministic layer ordering and bounded structured samples.

## Milestone Summary

**Key Decisions:**

- Adopt validate-before-mutate as the atomicity strategy.
- Reserve `tragicErr` for internal-invariant failures and recovered merge panics only.
- Replace the transformer-only failure taxonomy with unified `IngestFailureMode`.
- Keep `IngestError.Value` verbatim; callers own redaction and output-size policy.
- Defer SIMD parser implementation to v1.3 due upstream license, tag, and shared-library distribution blockers.

**Issues Resolved:**

- Ordinary public ingest failures no longer poison the builder.
- Parser, transformer, and numeric failure modes now share one configuration vocabulary.
- CLI continue-mode failures are machine-readable and grouped by failure layer.
- Phase 18 verification passed focused root tests, focused CLI tests, full `go test ./...`, and `make lint`.

**Issues Deferred:**

- SIMD parser adapter, datasets, benchmarks, and CI.
- `ValidateDocument` dry-run API.
- Snapshot-and-restore atomicity reserve strategy.
- Bloom `AddString` allocation cleanup and per-path `[*]` opt-out backlog items.
