# Quick Task 260422-lv1: PR #29 Review Feedback Fixes

## Goal

Resolve five targeted findings from the PR #29 review (Phase 14 — Observability
Seams). All changes are low-risk, internal, and backward-compatible.

## Scope

Items addressed (numbering from the PR review summary):

1. **Attribute vocabulary bug** — `adaptiveInvariantAllRGs` emits
   `operation=<EQ|NE|IN|NIN>` instead of the coarse boundary name. Swap to
   `AttrOperation(telemetry.OperationEvaluate)` + `AttrPredicateOp(op)`.
2. **`go mod tidy` drift** — OTel packages are `// indirect` in `go.mod` but
   imported directly by `telemetry/*.go`. Run `go mod tidy` and commit.
3. **Error-type constant duplication** — `normalizedErrorTypeOther` is
   declared in both `parquet.go` and `telemetry/boundary.go`. Promote to a
   single exported `telemetry.ErrorTypeOther`; update call sites in
   `parquet.go` and `serialize.go`.
4. **Allocs test name/comment mismatch** — `TestEvaluateDisabledLoggingAllocsZero`
   documents "0 allocations" but asserts `baseAllocs+1`. Update name, comment,
   and error text to reflect the actual `≤1 alloc` tolerance.
5. **Misleading test name** — `TestBuildFromParquetContextEmitsParserNameWithoutInfoLeak`
   promises to verify the parser-name info-leak constraint but only ensures
   the function doesn't panic. Wire a capturing logger and assert no
   parser-name attr at INFO level.

Items **not** addressed (explicitly deferred):

- Item 5 (Claude review `Info/Warn/Error` guard) — design choice, only a
  doc-comment nit.
- Item 6 (classifyParquetError string heuristics) — already sentinel-first;
  revisit if misclassification observed in the wild.
- Items 8, 9 (slogadapter context.Background, NewSignals(nil) Enabled()=true)
  — verified non-issues / already documented.
